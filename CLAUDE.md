# 硕士论文自动写作系统 — SimpleSentiment 舆情系统

## 项目背景

本论文基于 SimpleSentiment 项目 (https://github.com/cg-collector/SimpleSentiment)，
这是一个轻量级舆情监测分析预测系统，技术栈如下：

- **语言**: Python 3.9+
- **Web框架**: Streamlit
- **情感分析**: SnowNLP
- **时序预测**: Prophet
- **可视化**: Plotly + WordCloud
- **存储**: SQLite
- **LLM集成**: OpenAI API（可选增强模式）
- **部署**: Docker / 本地直接运行

### 系统核心模块
1. **数据采集层** (collectors/): 多源数据采集（新闻API、微博爬虫），含去重机制
2. **情感分析层** (analyzers/): SnowNLP情感分析 + 关键词提取 + 趋势分析
3. **预测层** (predictors/): Prophet时间序列预测 + LLM辅助深度分析
4. **展示层** (pages/): Streamlit多页面应用（监测/分析/预测/设置）
5. **工具层** (utils/): 数据库操作、LLM客户端、可视化、文本处理

### 论文题目
《基于多源数据融合的轻量级舆情监测分析与预测系统设计与实现》

## 📝 写作风格规范（极其重要）

### 必须做到
- **句式变化**：长短句交替，不要每句都20-30字
- **具体优于抽象**：不说"取得了良好效果"，要说"准确率从78.3%提升到91.7%"
- **用项目真实细节**：代码片段、配置参数、实际数据都是好素材
- **第一人称**：适当使用"本文""笔者""我们"
- **自然过渡**：不要每段都用"此外""另外""同时"
- **引用分散**：不要在一段里堆3-5个引用
- **段落长度不均匀**：有的3-4句，有的6-8句

### 绝对禁止
- ❌ 连续"首先…其次…再次…最后"
- ❌ 每段以"在…方面"或"从…角度"开头
- ❌ "具有重要的理论意义和实践价值"之类的空话
- ❌ "综上所述""总而言之"等AI高频词
- ❌ 每节结构都一模一样
- ❌ 每段末尾都来一句总结

### 舆情系统论文的特殊要求
- 提及具体的API（NewsAPI、天行数据）和数据源时要有技术细节
- 情感分析部分要有SnowNLP的数学公式和算法描述
- Prophet预测部分要有模型公式 y(t) = g(t) + s(t) + h(t) + ε
- 展示系统界面时用文字详细描述截图内容（因为docx中不插图）
- 代码展示要有上下文解释，不能贴了代码就跳到下一段

## 🔍 AIGC自评协议

每写完一章，以"AIGC检测专家"角色审查：

### 评分标准（每段0-10分）
- 句式单一性（同结构句子越多分越高）
- 过渡词密度（"此外/因此/然而"过多）
- 段落结构规律性（每段总分总）
- 用词空洞程度（抽象形容词多）
- 信息密度（言之无物）

### 判定
- 整体 < 25%：✅ 通过
- 25%-40%：⚠️ 改写高风险段
- > 40%：🔴 大幅重写

### 改写手段
1. 打乱句子结构
2. 抽象→具体（用项目真实数据/代码替换）
3. 删减过渡词
4. 段落长度制造不均匀感
5. 加入第一人称判断和项目开发中的实际经验

## 📄 DOCX 格式规范

### 页面
- 纸张：A4 (11906 x 16838 DXA)
- 页边距：上下 2.54cm(1440 DXA)，左 3cm(1701 DXA)，右 2cm(1134 DXA)
- 行距：1.5倍

### 字体
- 中文正文：宋体 12pt（小四）
- 英文/数字：Times New Roman 12pt
- 章标题（一级）：黑体 16pt 加粗，居中，前另起一页
- 节标题（二级）：黑体 14pt 加粗
- 小节标题（三级）：黑体 12pt 加粗
- 首行缩进：2字符（约480 DXA）
- 页码：页脚居中
